## 재시도 결과: 프록시 상황은 그대로 — CA 주입은 여전히 필요해요

CA 주입 없이 전부 다시 시도해봤지만 결과는 동일했습니다:

| 시도 | 결과 |
|------|------|
| node fetch → relay (CA 없이) | ❌ `SELF_SIGNED_CERT_IN_CHAIN` |
| npm registry ping | ❌ self-signed certificate 오류 |
| dev 서버 + curl (CA 없이) | ❌ `"릴레이 서버에 연결할 수 없어요"` |
| TLS 체인 확인 | 여전히 `lfcorp` 프록시가 가로채는 중 |
| dev 서버 + curl (**CA 주입**) | ✅ chunk 스트리밍 + done 이벤트 정상 |

회사 프록시(lfcorp)가 여전히 모든 외부 HTTPS를 가로채고 있어서, Node 프로세스는 `NODE_EXTRA_CA_CERTS` 없이는 relay에 연결할 수 없어요. curl이나 브라우저는 macOS 키체인을 쓰니 정상인데, Node만 자체 CA 번들을 써서 실패하는 구조라 네트워크가 바뀌지 않는 한 계속 필요합니다.

그래서 CA를 임시 폴더 대신 **`~/lfcorp-ca.pem`** 으로 영구 저장해뒀고, 지금 dev 서버를 이 CA와 함께 **localhost:3000에 실행 중**으로 남겨놨습니다 — 브라우저에서 바로 테스트해보실 수 있어요.

앞으로 로컬에서 relay 연동 앱(dream, health)을 돌릴 때는 이렇게 실행하시면 됩니다:

```bash
NODE_EXTRA_CA_CERTS=~/lfcorp-ca.pem pnpm dev
```

매번 붙이기 번거로우시면 `~/.zshrc`에 `export NODE_EXTRA_CA_CERTS=~/lfcorp-ca.pem` 한 줄을 추가하는 방법도 있어요 (npm/pnpm 다운로드 행업도 같이 해결됩니다). 원하시면 추가해드릴게요.
